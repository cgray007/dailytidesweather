# claudecode
Claude Code originated projects

## Daily Tide Email (C# / .NET 10)

Console app that emails a daily tide + weather report for 136th Street, Ocean City, MD.
Data comes from the NOAA Tides and Currents API (station 8570283 — Ocean City Inlet) and
Open-Meteo; the morning narrative is generated by a Claude Managed Agent.

| Project | Purpose |
|---|---|
| `TideEmail` | Daily report: fetches tides, water temp, and weather, generates the narrative via Claude, and sends the email through Gmail SMTP. Run by the `Daily Tide Email` GitHub Actions workflow. |
| `CreateAgent` | One-time setup: creates the "OC Tide Narrator" Managed Agent and its cloud environment, and prints the IDs to store as GitHub Secrets (`TIDE_AGENT_ID`, `TIDE_ENV_ID`). |

### Build & run

```sh
dotnet build DailyTidesWeather.slnx
dotnet run --project TideEmail
```

`TideEmail` requires these environment variables (set as GitHub Secrets for the workflow):
`ANTHROPIC_API_KEY`, `TIDE_AGENT_ID`, `TIDE_ENV_ID`, `GMAIL_ADDRESS`, `GMAIL_APP_PASSWORD`,
`RECIPIENT_EMAIL`. `CreateAgent` needs only `ANTHROPIC_API_KEY`.

### Local secrets

On a local run (anywhere `GITHUB_ACTIONS`/`CI` is not set), both apps sync those variables
with a per-user store at `%APPDATA%\DailyTidesWeather\secrets.json` (Linux/macOS:
`~/.config/DailyTidesWeather/secrets.json`, chmod 600):

- Any of the variables that **are** set in the environment are written to the file.
- Any that are **not** set are loaded from the file into the process.

So after one run with the variables exported, subsequent local runs need no environment
setup. In GitHub Actions the sync is skipped and secrets come only from the workflow env.

Emails are sent only during the season (May 17 – October 15).
